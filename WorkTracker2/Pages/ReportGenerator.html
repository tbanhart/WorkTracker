Welcome to the WorkTracker Documentation

Report Generator

This is the main window for viewing the details of letters which have had QC sheets generated. The Report Generator utility allows for profiles to be designated for generation to create lists of letter details and counts. All profile information is saved locally, meaning if the data is changed to have a new profile added, it will be removed when the program is updated to a new version.

Report Details

Generation of reports is covered in the Report Details section. A start date and end date can be selected to create a range of dates in which to include in the report that is generated. The details of the report are set using the Reporting Profile list. Each profile has an effect on the levels of the report tree that is created, meaning it is important to know which profile will include the information they are needing to find. Additionally, the profile sets which details of a letter will show counts in the tree.

Tables

Each profile has a Table that it relates to, so when selecting a profile, it is important to select the correct table in order to find the profile in the list. The Active Letters table of the database shows letters that have not yet been passed through QC or destroyed after submitting. When a letter is passed or destroyed, it is sent to the Archive table. To check the details of when a letter changed in status, there is also a QC History table to lookup status changes in a letter.

Generating Reports

When all fields in Report Details are complete, the Generate Report button will fill in the Report Tree with the information specified in the profile for the items submitted in the date range.

QC Controls

The Report Generator is built with a QC Controls implemented to track the current status of a letter through to QC completion. By selecting a correspondence number in the Report Tree and clicking a control button, a user doing QC can update the status of a letter and send it on to the next step. For Removing or Failing a letter, the user must designate a reason why. Common reasons are populated into the comments box, however if none of them matches, a user can enter a custom entry in order to make for a more accurate status update. When failing a letter, a new QC sheet will be printed to replace the old one so it accurately represents the status when handed back to the user that matched the letter. When approving a letter, no comment is necessary and the letter will be completed and archived.

Search

In order to find a specific letter or group of letters, a Search function has been created. Entering a correspondence number will cause the specified table to be searched for the number within the date frame selected and all information entered with it will be presented. If a value other than a correspondence number is entered, the generator will search for that term in the selected date frame to retrieve all letters related to that term and display their correspondence numbers. For example, this feature can be used to search all submitted letters for a specified user or client when attempting to find a correspondence number.

Report Tree

The Report Tree is the area that all generated reports are displayed in. When a report is generated, the Tree will be filled with nodes that can be expanded or collapsed to sort through the information about the letters more conveniently. When a correspondence number is selected and the user opts to have the number changed in QC status, the node will be removed, as to prevent applying multiple status changes to a letter. If the user wishes to view the item again, provided it is still in the specified table, the user may refresh the tree by generating it again.
